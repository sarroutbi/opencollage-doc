      \fontsize{5.5}{6.5}\selectfont
      \begin{itemize}
        \setlength{\itemsep}{0pt}
        \item AI agent frameworks (AutoGen, CrewAI, LangGraph) define
              \textbf{incompatible} agent interfaces
        \item No standard exists for an agent from one framework to participate
              as a node in another framework's workflow
        \item Existing orchestration (Kubernetes, Airflow) manages
              \textbf{containers/tasks}, not \textbf{AI agents} with typed
              handoffs and telemetry
        \item Each integration requires bespoke glue code---$O(n^2)$ adapters
              for $n$ frameworks
      \end{itemize}
    \end{alertblock}

    \begin{block}{\fontsize{5.5}{6.5}\selectfont Novel Contribution}
      \fontsize{5.5}{6.5}\selectfont
      A \textbf{minimal, universal adapter contract} (three methods) that any AI
      agent framework implements to become a first-class node in a shared
      orchestration DAG:
      \begin{enumerate}
        \setlength{\itemsep}{0pt}
        \item \texttt{initialize(config)} --- bootstrap from declarative YAML
        \item \texttt{execute(input, checkpoint)} --- run with typed I/O and
              state checkpointing
        \item \texttt{emit\_telemetry(span)} --- report execution metrics via
              OpenTelemetry
      \end{enumerate}
      The contract reduces $O(n^2)$ integrations to $O(n)$ adapters.
    \end{block}

        \item Enforces \textbf{typed handoffs} via schema references validated
              at pipeline load time
        \item Supports \textbf{human-in-the-loop gates} as first-class DAG nodes
        \item Is \textbf{portable}: same spec runs locally, in CI, or on

  \vspace{-0.3cm}

  \fontsize{5.5}{6.5}\selectfont
  \begin{columns}[T]
    \column{0.44\textwidth}
    \begin{alertblock}{\fontsize{5}{6}\selectfont Problem (Prior Art)}
      \fontsize{5}{6}\selectfont
      \begin{itemize}
        \setlength{\itemsep}{0pt}
        \item Each agent framework manages state \textbf{independently} and \textbf{incompatibly}:\\
          {\fontsize{4.5}{5.5}\selectfont
          \textbullet\ LangGraph: \texttt{StateGraph} with typed state channels\\
          \textbullet\ AutoGen: chat history as implicit state\\
          \textbullet\ CrewAI: task context passed between crew members}
        \item No mechanism exists to \textbf{checkpoint} the state of a
              multi-framework pipeline at inter-agent boundaries
        \item If a pipeline fails mid-execution, there is no way to
              \textbf{replay from a checkpoint} across different frameworks
        \item Debugging multi-agent pipelines requires reproducing the
              \textbf{entire execution}, not just the failed step
        \item Compliance and audit requirements demand \textbf{deterministic
              replay} of agent decision chains
      \end{itemize}
    \end{alertblock}
    \vspace{-0.2cm}
    \begin{block}{\fontsize{5}{6}\selectfont Novel Contribution}
      \fontsize{5}{6}\selectfont
      A \textbf{cross-framework state management protocol} that:
      \begin{enumerate}
        \setlength{\itemsep}{0pt}
        \item Defines a \textbf{universal checkpoint format} capturing the typed
              handoff payload, agent config, and execution metadata at each
              DAG edge
        \item Stores checkpoints in a \textbf{pluggable state backend}
              (Redis, PostgreSQL, S3) independent of any framework
        \item Enables \textbf{replay from any checkpoint}: re-execute the
              pipeline from a specific node with the exact same inputs
        \item Supports \textbf{time-travel debugging}: inspect the state at
              any point in a past execution
        \item Provides \textbf{audit trail}: immutable log of all state
              transitions with timestamps and agent identifiers
      \end{enumerate}
    \end{block}

        \item \textbf{vs.\ MLflow experiment tracking}: tracks model

        \item \textbf{Novel}: cross-framework checkpoint/replay with
